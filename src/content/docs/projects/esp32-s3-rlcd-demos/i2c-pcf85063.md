## 一句话目标

跑通 `04_I2C_PCF85063`，看懂 ESP32-S3 如何通过 I2C 和 PCF85063 RTC 通信，并完成时间写入和读取。

## 先懂概念

I2C 是一种常见的两线通信方式。两线通常叫 `SCL` 和 `SDA`：`SCL` 像节拍器，负责时钟；`SDA` 负责传数据。同一条 I2C 总线上可以挂多个设备，每个设备靠地址区分。

RTC 是 Real-Time Clock，中文常叫“实时时钟”。它像一块专门负责计时的小表，即使主控 ESP32-S3 重启了，也可以继续保存年月日时分秒。

PCF85063 就是一颗 RTC 芯片。这个 demo 里，它的 I2C 地址是 `0x51`。ESP32-S3 通过 I2C 给它写入时间，再每秒把时间读出来打印。

备份电池是 RTC 很重要的概念：如果板子给 RTC 接了纽扣电池或其它备用电源，主电源断开时 RTC 还能继续走时。没有备份电池时，断电后时间可能丢失或需要重新设置。

## 硬件/代码入口

事实来源目录：

`D:\Tools\ESP-IDF\ESP32-S3-RLCD-4.2-Demo\ESP32-S3-RLCD-4.2-Demo\02_ESP-IDF\04_I2C_PCF85063`

你最该看的 4 个入口：

- `main/main.cpp`：`app_main()` 只调用 `UserApp_AppInit()`。
- `components/user_app/user_app.cpp`：建立 I2C 总线、设置 RTC 时间、创建读取任务。
- `components/port_bsp/i2c_bsp.cpp`：封装 I2C master bus 的读写能力。
- `components/port_bsp/i2c_equipment.cpp`：把 PCF85063 接到 I2C，并封装 `Rtc_SetTime()`、`Rtc_GetTime()`。

demo 使用的 I2C 引脚是：

```cpp
I2cMasterBus I2cbus(14, 13, 0);
```

也就是 `SCL = GPIO14`，`SDA = GPIO13`，I2C port 为 `0`。PCF85063 地址是 `0x51`。

## 运行现象

烧录运行后，串口先打印：

```text
rtc-example run
```

如果 RTC 初始化成功，会看到类似：

```text
InitWill
```

随后每秒打印一次时间：

```text
2025/9/9 20:15:30
2025/9/9 20:15:31
```

这个 demo 在启动时会调用 `Rtc_SetTime(2025,9,9,20,15,30)`，所以每次重新运行都会先把 RTC 设置到这个时间，再开始读。

## 核心流程

核心流程可以直接读成一句话：

```text
app_main -> UserApp_AppInit -> Rtc_Setup -> Rtc_SetTime -> Rtc_LoopTask -> Rtc_GetTime
```

更细一点：

1. 全局创建 `I2cMasterBus I2cbus(14, 13, 0)`。
2. `UserApp_AppInit()` 打印启动信息。
3. `Rtc_Setup(&I2cbus, 0x51)` 把 PCF85063 加入 I2C 总线。
4. `Rtc_SetTime(2025,9,9,20,15,30)` 写入初始时间。
5. `Rtc_LoopTask()` 每秒调用 `Rtc_GetTime()`。
6. 串口打印年月日时分秒。

## 关键代码讲解

I2C 总线初始化在 `I2cMasterBus` 构造函数里完成。里面会配置：

```cpp
scl_io_num
sda_io_num
i2c_new_master_bus(...)
```

这表示 ESP32-S3 作为 I2C master，负责发起读写。

添加 PCF85063 设备在 `Rtc_Setup()` 里完成：

```cpp
i2c_master_bus_add_device(...)
```

这个 demo 给 PCF85063 设置的地址是 `0x51`，速度是 `300000` Hz。地址必须对，否则读写都会失败。

RTC 库初始化用的是：

```cpp
rtc.begin(I2cDevCallback)
```

这里的 `I2cDevCallback` 是一个桥：PCF85063 库想读写寄存器时，会通过这个 callback 调到 demo 自己的 I2C 读写函数。

写时间很短：

```cpp
Rtc_SetTime(...)
```

底层实际调用 `rtc.setDateTime(...)`，把年月日时分秒写进 PCF85063 的时间寄存器。

读时间也很短：

```cpp
Rtc_GetTime(...)
```

底层实际调用 `rtc.getDateTime()`，再把 year、month、day、hour、minute、second、week 填到 `rtcTimeStruct_t`。

## 动手改一改

适合零基础练手的改动：

1. 把 `Rtc_SetTime(2025,9,9,20,15,30)` 改成当前时间，观察串口从你设置的时间开始走。
2. 把 `vTaskDelay(pdMS_TO_TICKS(1000))` 改成 `5000`，观察打印间隔变成 5 秒。
3. 注释掉 `Rtc_SetTime(...)` 后重新上电，观察 RTC 是否还能保持时间；这可以帮助你判断板子上的 RTC 备份供电是否有效。

如果你只是想验证“读写通了没有”，先不要改 I2C 引脚和地址。

## 常见坑

- `SCL` 和 `SDA` 接反：I2C 不会正常通信。
- 地址写错：PCF85063 在这个 demo 里是 `0x51`，不是随便填。
- 每次启动都重新设时间：demo 的 `Rtc_SetTime(...)` 会覆盖 RTC 当前时间，所以测试掉电保持时要注意它。
- 没有备份电池却期待断电计时：RTC 要继续走时，需要对应的备用供电条件。
- 把 RTC 当成网络时间：RTC 只负责本地走时，不会自动联网校时。

## 和 Pixel Soul 项目的关系

Pixel Soul 如果需要日程、待机显示、番茄钟、睡眠记录或“设备重启后仍知道当前时间”，RTC 就是基础能力。

这个 demo 提供了最小可用路径：I2C 初始化、PCF85063 设备地址、时间写入、时间读取。移植到 Pixel Soul 时，建议保持模块边界清楚：底层 RTC 模块只提供 `set_time` 和 `get_time` 这类能力，上层业务再决定什么时候校时、什么时候显示、什么时候用时间触发状态变化。

还要记住：RTC 能保持时间，不等于它能知道真实世界当前时间。第一次使用、掉电丢时或时间漂移后，仍需要通过用户设置、网络时间或其它来源校准。
